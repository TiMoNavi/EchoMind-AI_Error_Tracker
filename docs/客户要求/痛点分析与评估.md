## 评估维度说明

| 维度 | 含义 | 评分 |
|------|------|------|
| **痛感** | 用户日常多难受（学生/家长实际感受） | 1-5，5=极痛 |
| **付费** | 有人会为解决这个问题单独花钱吗 | 1-5，5=强烈付费意愿 |
| **感知价值** | 用户看到解决方案时觉得"这个厉害/有用" | 1-5，5=一看就想要 |
| **实现难度** | 技术上做出来有多难 | 1-5，5=非常难 |
| **独特优势** | 竞品能不能轻易抄/替代 | 1-5，5=极难抄 |
| **竞品位置** | 我们在市场上的位置 | 唯一/领先/之一/落后 |

### 两个复合维度（用于二维矩阵）

- **需求得分** =（痛感 + 付费 + 感知价值）÷ 3 → **这个问题值不值得解决**
- **优势得分** =（独特优势 +（6 - 实现难度））÷ 2 → **我们解决这个问题有没有优势**

## 四、卖点候选总结

### 从第一象限中提取可包装为"卖点"的组合

> 单个痛点不是卖点。卖点 = 一组痛点的打包解决方案，能用一句话讲清楚。

**候选卖点 A：「像看电影一样清晰地看到掌握状态」**
打包痛点：#5 + #27 + #28 + #30
- 学生：一眼看到哪些会了、哪些不会、卡在哪一层
- 家长：随时知道孩子什么情况，不用再问老师
- 需求得分：5.0（顶格）
- 核心交付物：掌握度面板（L0-L5 可视化）+ 家长看得懂的表述
- 数据依赖：需要有掌握度数据来源（上传标对错 → 关联模型/知识点 → 计算Level）

**候选卖点 B：「高考题号视角——从来没有人这么做过」**
打包痛点：#20 + #23 + #24 + #34
- 按高考题号看自己的掌握程度：第3题你能不能拿到分？
- 卷面策略：你要考70分，哪些题必须拿、哪些可以放
- 需求得分：4.3-5.0
- 竞品位置：**唯一**（没有人做过）
- 核心交付物：卷面策略表 + 题号聚合面板 + 预测得分叠加
- 数据依赖：需要地区卷面结构模板 + 题号→模型映射 + 学生做题数据

**候选卖点 C：「不只告诉你错了，告诉你错在第几步」**
打包痛点：#1 + #10 + #11 + #12 + #16
- 四层诊断：建模→列式→执行→粗心，精确定位
- 做错题不再是"重做一遍"，而是"练你卡住的那一步"
- 需求得分：3.7-4.7
- 竞品位置：**唯一**（核心壁垒）
- 核心交付物：AI诊断引擎 + 分层训练流程
- 数据依赖：需要AI提示词 + 教研预设内容 + 模型/知识点标签体系
- ⚠️ 实现难度最高

## 五、三个卖点的依赖关系

```
卖点A（掌握状态可视化）         卖点B（题号视角+卷面策略）
      ↓ 需要数据                    ↓ 需要数据
      └──────┐         ┌────────────┘
             ↓         ↓
        【数据引擎层】（基础设施）
        - 模型/知识点标签体系
        - 题号→模型映射表
        - 掌握度计算规则(L0-L5)
        - 学生做题数据(上传+标对错)
             ↑
             │ 提供精确诊断数据
        卖点C（四层诊断+分层训练）
        ⚠️ 实现难度最高
        ⚠️ 但也是最核心壁垒
```

### 关键发现

**卖点A和B可以先用"简化版数据引擎"跑起来：**
- 学生上传卷子 → 标对错 → 系统知道每题对应什么模型 → 计算Level → 展示面板
- 不需要AI诊断（错题先标"待诊断"，Level用简化规则计算）
