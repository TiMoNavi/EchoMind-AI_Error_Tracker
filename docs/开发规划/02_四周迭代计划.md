# EchoMind 1个月MVP开发计划

> 3人纯开发团队 | 4周冲刺 | 教研数据由专人提供
> 目标：跑通"注册→诊断→状态展示→上传→推荐"主线

---

## MVP范围定义

### ✅ V1必做（本月）

| 功能域 | 范围 |
|--------|------|
| 域1-注册 | 三步注册流程（基础信息→卷面策略→简化诊断） |
| 域5-上传 | 拍照上传 + 手动标对错 + 上传历史 |
| 域4-诊断 | AI错题诊断（四层定位 + 三段式话术） |
| 域6-状态引擎 | 标签更新算法（升降级规则） |
| 域9-推荐 | 主页推荐列表（简化版，基于Level排序） |
| 域11-架构 | Tab导航 + 页面骨架 + 路由 |
| 域12-可视化 | 卷面热力图（简化版） |
| 全局知识/模型树 | 三级树浏览（只读展示） |
| Profile | 个人信息 + 目标分数 + 学习统计 |

### ❌ V1不做（下月）

| 功能域 | 原因 |
|--------|------|
| 域2-知识点学习5步 | 需要大量教研内容预设 |
| 域3-模型训练6步 | 最复杂的交互，依赖域6稳定 |
| 域7-闪卡 | 最独立，可后做 |
| 域8-预测中心 | 依赖数据积累 |
| 域9-周复盘 | 需要一周数据 |
| 社区Tab | 已确认砍掉 |

---

## 第1周：地基

**目标：后端能跑、数据库能存、Flutter能联调**

| 任务 | 产出 |
|------|------|
| PostgreSQL数据库Schema设计 + 建表 | 8张核心表 |
| FastAPI项目骨架 | 项目结构、中间件、错误处理、CORS |
| 用户认证API | 注册/登录/Token刷新 |
| Flutter网络层 | Dio封装、Token管理、API基类 |
| Flutter状态管理基础 | Riverpod Provider结构 |
| 注册页R1（基础信息） | 地区选择 + 科目 + 目标分数 |
| 定义教研数据交付格式 | JSON Schema模板→交给教研人员 |

**外部依赖：** 教研人员本周末前交付知识点清单 + 模型清单（JSON）

**里程碑：Flutter能注册账号并登录**

---

## 第2周：数据入口

**目标：能拍照上传题目、能看到知识树**

| 任务 | 产出 |
|------|------|
| 注册页R2（卷面策略展示） | 根据目标分+地区生成策略表 |
| 卷面策略生成引擎 | 纯规则，读取地区模板 |
| 状态引擎核心（标签更新算法） | 4条更新路径的规则引擎 |
| 上传功能（拍照+标对错） | 相机/相册 → 上传 → 手动标注 |
| 上传历史页 | 按日期分组的上传记录列表 |
| 全局知识点树页 | 三级树浏览（章→节→知识点） |
| 全局模型树页 | 三级树浏览（章→节→模型） |
| 种子数据导入脚本 | 教研交付JSON → 写入DB |

**外部依赖：** 教研人员本周中前交付天津物理地区模板（至少70/80分两档）

**里程碑：能上传题目、能浏览知识体系**

---

## 第3周：AI诊断 + 状态引擎

**目标：上传的错题能被AI诊断，状态能更新**

| 任务 | 产出 |
|------|------|
| AI诊断API（大模型调用） | 四层定位 + 三段式输出 |
| AI诊断页面 | 诊断结果展示（问题→可解决→分数影响） |
| 推荐排序算法 | 简化版priority_score |
| 主页推荐列表 | 基于Level的推荐排序 |
| 主页顶部仪表盘 | 今日学习数据 + 掌握概览 |
| 模型详情页 | 掌握度仪表盘 + 前置知识点列表 |
| 知识点详情页 | 状态仪表盘 + 关联模型 |

**里程碑：错题能诊断、Level能变化、主页有推荐**

---

## 第4周：可视化 + 注册诊断 + 打磨

**目标：注册诊断闭环、卷面热力图、整体可用**

| 任务 | 产出 |
|------|------|
| 注册页R3（简化AI诊断） | 3-5道题的自适应诊断 → 初始Level |
| 卷面热力图 | 按题号展示掌握状态的色块图 |
| 全局高考Tab | 按年份/题号浏览真题 |
| Profile页 | 用户信息 + 目标分 + 学习统计 |
| 上传菜单弹窗 | 作业/练习/考试三种上传入口 |
| 全链路联调 + Bug修复 | 注册→诊断→上传→推荐 全流程 |
| 部署 | Docker打包 + 服务器部署 |

**外部依赖：** 教研人员第3周末前交付注册诊断预设题 + 高考题号→模型映射

**里程碑：完整MVP可演示**
